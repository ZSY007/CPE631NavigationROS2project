Nav2 navigation：开是什么
19:16

Nav2 navigation：开 的意思是启动 ROS2 的 Navigation2 导航栈，让机器人真的在 Gazebo 里按目标点移动。

具体包括这些核心节点：

map_server：加载 maps/cafe.yaml
amcl：定位机器人
planner_server：全局路径规划，这里分别用 A* 或 D*
controller_server：局部控制器，跟随路径并发布 /cmd_vel
bt_navigator：接收 NavigateToPose 目标并调度规划/控制
local_costmap / global_costmap：维护障碍物代价地图
所以你的实验不是离线算路径，而是：

Gazebo 机器人 + 动态行人
-> Nav2 接收 goal_sender 的目标点
-> A* 或 D* 算全局路径
-> MPPI controller 控制机器人走
-> data_collector 记录结果
只是这次 social_navigation=false，所以没有 /social_costmap 那层社交代价。





19:16
那为什么趋势线还是可以避障的
19:17

因为不开 social 不等于不避障。

你现在关掉的是 /social_costmap，也就是“预测式社交代价层”：提前考虑行人未来位置、社交距离、个人空间这些。

但 Nav2 本身还有普通避障：

/scan 激光雷达
-> local_costmap obstacle_layer / voxel_layer
-> MPPI controller
-> 调整 /cmd_vel 绕开障碍
所以机器人仍然会避开看到的障碍物和动态行人，只是这种避障是“反应式”的：行人进入激光雷达/局部代价地图后，局部控制器才绕开或减速。

区别可以这样写：

dynamic_astar / dynamic_dstar：普通动态避障，有行人，有 /scan，无社交预测。
social_astar / social_dstar：普通动态避障 + /social_costmap，会提前考虑行人运动趋势和社交距离。
所以你看到轨迹线会绕开障碍是正常的，那是 Nav2 local costmap 和 MPPI 在工作，不是 social navigation。
