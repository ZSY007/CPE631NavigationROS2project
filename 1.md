## 文件结构约定

- `cpe631_ros2/`：主包，尽量保持原始状态，用于仿真运行。
- `codefiles/`：自己写的改进版代码，提交作业主要看这里。
- `param/`：Nav2 配置文件，尽量保持原始状态；`nav2_social_smac.yaml` 是 Smac Hybrid A* + local/global social costmap 的新增实验配置。
- `launch/`：启动文件，实验脚本会同步到 ROS2 workspace。

重要规则：修改尽量写进 `codefiles/`，不要直接改原始备份目录。

如果要在 ROS2/Nav2 仿真中使用 `codefiles/` 里的改进版节点，先复制到主包，再编译：


## 每个核心文件说明

### `ped_pose_extractor.py`

用途：从 Gazebo `/world/cafe/dynamic_pose/info` 或桥接后的模型位姿中提取行人位姿，输出 `/pedestrian_poses` 或调试 topic。

主要用于给 `social_nav_node.py` 和 `data_collector.py` 提供行人位置。

### `social_nav_node.py`

用途：实现 Anisotropic Gaussian Predictive Social Costmap。

流程：

```text
/pedestrian_poses
-> 行人速度估计
-> 行人未来位置预测
-> 各向异性高斯社交代价
-> /social_costmap
-> Nav2 local/global costmap
```

它让机器人提前避开行人的运动方向和未来可能位置，而不是只避开当前障碍。

### `data_collector.py`

用途：记录实验性能指标并写入 CSV。

主要指标：

| 字段 | 含义 |
| --- | --- |
| `path_length_m` | 路径长度 |
| `time_s` | 到达目标所需时间 |
| `encounters` | 接近行人次数 |
| `personal_violations` | 进入个人空间次数 |
| `intimate_violations` | 进入亲密空间次数 |
| `min_ped_dist_m` | 与行人最近距离 |
| `mean_angular_acceleration` | 平均角加速度，越小越平滑 |

重要时间口径：

- ROS2/Nav2 仿真实验里的 `data_collector.py` 使用 `use_sim_time:=true`。
- CSV 的 `time_s` 是 ROS/Gazebo 仿真时间，不是实际墙钟时间。
- `goal_sender.py` 的 `EXPERIMENT_TIMEOUT` 也按 ROS/Gazebo 仿真时间计算；`EXPERIMENT_WALL_TIMEOUT` 才是真实墙钟看门狗。
- 因此，用 `4.csv` 做算法性能比较时，应统一比较 `time_s` 仿真秒。
- 如果单独讨论“实际机器跑实验花了多久”，再用实际 real-time factor (RTF) 单独换算墙钟时间，不要把墙钟换算值写回 `4.csv` 的 `time_s`。

### `peds.py`

用途：生成和控制行人，包括多个直线往返行人和 1 个随机游走行人，并发布 `/pedestrian_poses`。

### `goal_sender.py`

用途：通过 Nav2 `NavigateToPose` action 依次发送多个目标点。

单独运行示例：

```bash
source /home/dx/ros2_ws/A/install/setup.bash
ros2 run cpe631_ros2 goal_sender
```

自定义路线示例：

```bash
ros2 run cpe631_ros2 goal_sender --ros-args \
  -p route:="0.5,-6.5,2.08;-3.5,0.0,0.89;2.5,6.0,-2.75;-2.0,2.1,-1.10;2.0,-4.0,-2.16"
```

### `integration_guide.yaml`

用途：说明如何把 `/social_costmap` 作为 Nav2 costmap 的一层接进去。它是集成说明文件，不是直接运行的 launch 文件。

## ROS2 三组/四组实验怎么跑

自动实验脚本：

```bash
cd /home/dx/CPE631-Navigation-ROS2-main
chmod +x codefiles/run_all_experiments.sh
./codefiles/run_all_experiments.sh
```

默认实验模式在脚本中定义，包括：

- `baseline`：无行人，无 social。
- `dynamic`：有行人，普通动态避障，无 social。
- `social`：有行人，Dijkstra/NavFn + 社交代价图。
- `social_astar`：有行人，A* + 社交代价图。
- `social_smac`：有行人，Smac Hybrid A* + local/global 社交代价图。
- `dynamic_dstar`：有行人，D* planner，不开 social。
- `dstar_social`：有行人，D* planner + 社交代价图。

只跑某些模式：

```bash
MODES=dynamic,social RUNS_PER_MODE=5 ./codefiles/run_all_experiments.sh
```

每组跑 25 轮：

```bash
RUNS_PER_MODE=25 \
CSV_FILE=/home/dx/ros2_ws/A/experiment_results_25run.csv \
./codefiles/run_all_experiments.sh
```

### 当前汇总结果文件 `4.csv`

当前根目录的 `4.csv` 是主要 Nav2 仿真实验汇总表，包含多组最终实验数据：

- `baseline`
- `dynamic`
- `dynamic_astar`
- `dynamic_dstar`
- `social`
- `social_astar`
- `dstar_social`

其中 `dstar_social` 来自单独跑的 2x physics target 批次：

```bash
RUNS_PER_MODE=25 \
MODES=dstar_social \
MAX_RETRY=1 \
EXPERIMENT_TIMEOUT=240 \
EXPERIMENT_WALL_TIMEOUT=900 \
ENABLE_RVIZ=0 \
GZ_GUI=0 \
CSV_FILE=/home/dx/ros2_ws/A/dstar_social_2x.csv \
./codefiles/run_all_experiments.sh
```

注意：只有这一组 `dstar_social_2x.csv` 改过 Gazebo physics 目标速度。设置为 2x 后，机器实际没有跑到 2x；根据日志估算，实际 RTF 约为 `0.843x`。这会影响真实墙钟耗时，但不改变 CSV 里的仿真时间口径。

`4.csv` 中的 `dstar_social` 行已经使用原始 CSV 的 `time_s` 仿真时间，而不是墙钟换算时间。这样它可以和前面没有改 physics 目标速度的几组一起做算法结果比较。

本次整理时曾误把 `dstar_social` 的 `time_s` 换算成旧 `0.3x` 墙钟时间写入 `4.csv`，后来已恢复为原始仿真时间。相关备份：

```text
4.csv.bak_20260430_170547
4.csv.bak_before_restore_sim_time_20260430_170821
```

### `dstar_social` 2x 批次墙钟换算

单独分析运行耗时时，使用：

```bash

  -o /home/dx/ros2_ws/A/dstar_social_2x_time_equiv.csv \
```

输出文件：

```text

```

这份文件只用于说明真实运行耗时，不用于替换 `4.csv` 的 `time_s`。

当前摘要：

| 指标 | 数值 |
| --- | --- |
| 最终轮数 | 25 |
| 成功轮数 | 21 |
| 平均仿真时间 `time_s` | 124.88 s |

报告里可以这样写：

```text
actual achieved RTF was about 0.843x. Therefore, its time_s values remain
simulation-time values for algorithm comparison, while wall-clock runtime is
reported separately using the measured RTF.
```

## A* 与 D* Lite 算法对比怎么跑

这部分不启动 Gazebo/Nav2，不开 social navigation，只在 `maps/cafe.yaml` 上比较 A* 和 D* Lite 算法本身。默认会使用和 `codefiles/peds.py` 一样的 5 个初始行人位置，并把它们投影成地图上的动态障碍：

```text
(3.0, 5.0), (-3.0, 5.0), (2.0, -4.5), (0.0, 6.0), (-1.0, 2.0)
```

A* 会直接在带行人障碍的地图上规划。D* Lite 会先在原地图上规划，再把同一批行人障碍作为动态更新加入并重规划。

代码文件：

```bash
codefiles/dstar_lite_planner.py
```

单次运行：

```bash
cd /home/dx/CPE631-Navigation-ROS2-main
python3 codefiles/dstar_lite_planner.py --map maps/cafe.yaml --algorithm both
```

不使用行人时：

```bash
python3 codefiles/dstar_lite_planner.py --map maps/cafe.yaml --algorithm both --pedestrians none
```

动态障碍重规划示例：

```bash
python3 codefiles/dstar_lite_planner.py \
  --map maps/cafe.yaml \
  --start 1.0,0.0 \
  --goal 2.0,-4.0 \
  --algorithm dstar \
  --block 150,181
```

只跑 A* 和 D* Lite 两个算法各 25 轮，并输出 CSV：

```bash
cd /home/dx/CPE631-Navigation-ROS2-main
RUNS_PER_ALGO=25 \
CSV_FILE=/home/dx/CPE631-Navigation-ROS2-main/a_dstar_results_25.csv \
./codefiles/run_ad_experiments.sh
```

自定义行人位置和行人障碍半径：

```bash
RUNS_PER_ALGO=25 \
PEDESTRIANS="3.0,5.0;-3.0,5.0;2.0,-4.5;0.0,6.0;-1.0,2.0" \
PED_RADIUS=0.35 \
CSV_FILE=/home/dx/CPE631-Navigation-ROS2-main/a_dstar_results_25.csv \
./codefiles/run_ad_experiments.sh
```

CSV 输出列：

| 列名 | 含义 |
| --- | --- |
| `experiment` | 第几轮 |
| `mode` | `astar` 或 `dstar_lite` |
| `result` | 是否找到路径，通常为 `succeeded` |
| `attempt` | 尝试次数，A*/D* 独立规划固定为 `0` |
| `path_length_m` | 路径长度，单位 m |
| `time_s` | 算法规划耗时，单位 s |
| `encounters` | 最终路径进入行人 `1.0 m` 范围的次数 |
| `personal_violations` | 最终路径进入行人 `1.2 m` 个人空间的次数 |
| `intimate_violations` | 最终路径进入行人 `0.45 m` 亲密空间的次数 |
| `min_ped_dist_m` | 最终路径与行人的最小距离 |
| `mean_angular_acceleration` | A*/D* 独立规划无 `/cmd_vel`，填 `0` |

## 长时间实验和重启脚本

### `run_sequential_reboot.sh`

用途：按顺序跑多个模式，中间自动重启机器，避免 Gazebo/Nav2 长时间运行导致状态污染。

默认顺序：

```text
dynamic -> social -> social_astar
```

查看状态：

```bash
./codefiles/run_sequential_reboot.sh --status
```

重置状态：

```bash
./codefiles/run_sequential_reboot.sh --reset
```

### `setup_and_start.sh`

用途：配置 `@reboot` 自动继续实验，并启动第一轮。这个脚本会改 sudoers 和 crontab，所以需要 sudo。

运行：

```bash
sudo bash codefiles/setup_and_start.sh
```

监控日志：

```bash
tail -f /home/dx/ros2_ws/A/run_logs/master.log
```
